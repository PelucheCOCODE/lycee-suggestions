      Calibration détails

      Ces exemples indiquent à l'IA si une suggestion manque de détails (lieu, impact) ou si elle est déjà précise / apporte un rapport. Un rapport = précisions qui complètent un sujet existant (ex: « fenêtres bat C » puis « surtout salle C203 »). Les détails s'accumulent dans la description, le titre reste général.

      Prompt pour ChatGPT / IA externe

      Copiez ce prompt et envoyez-le à ChatGPT (ou autre IA). L'IA répondra un fichier JSON que vous pouvez importer ci-dessus. Le prompt sert à savoir si une suggestion a besoin de détails en plus (lieu, impact, etc.).

      Calibration rapport & précision

      Ces exemples entraînent l'IA à détecter si une nouvelle suggestion a un RAPPORT avec une existante (même sujet) et si c'est une PRÉCISION (ajoute des détails : lieu, salle, etc.) ou pas. Rapport=non → nouvelle suggestion. Rapport=oui + précision=oui → demander à l'élève. Rapport=oui + précision=non → simple vote.

      Prompt pour ChatGPT / IA externe

      Génère des exemples (existing_text, new_text, has_rapport, is_precision) pour entraîner l'IA.

      Horaires bus

      Configuration de l'affichage des horaires STRAN sur les écrans. Les bus s'affichent aux créneaux définis ou en mode forcé.

      Configuration

      Collez la clé seule ou l'URL complète (ex: ...?apiKey=xxx), la clé sera extraite.
      Utilise le flux GTFS statique au lieu du temps réel. Les bus sont souvent à l'heure.

      Pages à exclure

      Sélectionnez les écrans /tv/... à exclure de l'affichage bus (la page Boîte à Idées n'est pas concernée).

      Mode test

      Créneaux d'affichage

      Format HH:MM. Pendant ces créneaux, les bus s'affichent sur les écrans.
